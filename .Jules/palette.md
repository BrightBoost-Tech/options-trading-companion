## 2024-07-22 - WAI-ARIA Disclosure Patterns in Dashboard Collapsibles
**Learning:** Custom interactive elements, like native `<button>` tags used to toggle dashboard list sections (e.g., Staged, Blocked queues in TradeInbox), frequently lack proper accessibility patterns out-of-the-box. Without `aria-expanded`, `aria-controls`, and `focus-visible` ring styles, these interactive elements are severely degraded for keyboard and screen-reader users, violating basic WCAG principles.
**Action:** When working on complex dashboard panels with custom collapsible sections, always verify that native toggle triggers implement the WAI-ARIA disclosure pattern (`aria-expanded`/`aria-controls`) and define explicit CSS utilities for keyboard focus visibility (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring`).
