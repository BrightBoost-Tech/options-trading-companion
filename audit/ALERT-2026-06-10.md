# ALERT — CRITICAL — 2026-06-10 (v5 FULL baseline run, Area 1)

## Debit-spread PoP/EV is computed from RAW LONG-LEG DELTA — live entries are being admitted on sign-flipped EV math

**Nothing was changed by this audit (read-only contract). This file exists so the human can act in the morning. Full report: `audit/reports/2026-06-10.md`.**

### WHAT

The strategy class behind ~all recent live fills (debit spreads) computes probability-of-profit as `abs(long-leg delta)`, not probability of clearing breakeven. The codebase's own intended correction (breakeven interpolation in `calculate_pop`, comment: "breakeven = long_strike + net_debit, so PoP < delta of long leg") is **unreachable from the only production call path**, for two independent reasons:

1. `packages/quantum/options_scanner.py:3411-3418` calls `calculate_ev` **without `legs`** (`:3409` sets `delta_for_ev` = abs(long-leg delta) for debit structures).
2. `packages/quantum/ev_calculator.py:165` passes `credit=premium` **only** for `("credit_spread","short_call","short_put")` — debit spreads get `credit=None`, so even with legs the premium-fraction interpolation would not engage. (Also a key mismatch: scanner legs use `'side'`, `calculate_pop` expects `'action'`.)

What actually runs: `ev_calculator.py:91-92` → `return abs(float(delta))`. The bias is maximal exactly where the builder selects (ITM long legs, premium_fraction ≈ 0.5), so the bug also **steers strike selection toward the worst-overstated structures**. The raw delta is persisted as `probability_of_profit` (`options_scanner.py:3865-3884`) and consumed by every downstream profit decision: the execution-cost-vs-EV gate (`options_scanner.py:3661`), the $15 MIN_EDGE gate + ranking (`canonical_ranker.py:40-56`), allocation skew, and the calibration training target (`workflow_orchestrator.py:3540/:3597`).
