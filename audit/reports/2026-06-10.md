## 1. ⚠ ALERT — CRITICAL (also written to `audit/ALERT-2026-06-10.md`)

### Debit-spread PoP/EV is computed from RAW LONG-LEG DELTA — live entries are being admitted on sign-flipped EV math

**The strategy class behind ~all recent live fills computes probability-of-profit as `abs(long-leg delta)`, not probability of clearing breakeven.** The codebase's own intended correction (breakeven interpolation in `calculate_pop`, with the comment "breakeven = long_strike + net_debit, so PoP < delta of long leg") is **unreachable from the only production call path**, for two independent reasons:

1. `options_scanner.py:3411-3418` calls `calculate_ev` **without `legs`** (`:3409` sets `delta_for_ev` = abs(long-leg delta) for debit structures);
2. `ev_calculator.py:165` passes `credit=premium` **only** for `("credit_spread","short_call","short_put")` — debit spreads get `credit=None`, so even with legs the premium-fraction interpolation would not engage. (Note also a key mismatch: scanner legs use `'side'`, `calculate_pop` expects `'action'`.)

Result: `ev_calculator.py:91-92` `return abs(float(delta))` is what runs. The bias is maximal exactly where the builder selects (ITM long legs, premium_fraction ≈ 0.5), so the bug also **steers strike selection toward the worst-overstated structures**.
